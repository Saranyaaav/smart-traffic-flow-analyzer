Frontend Setup

Android application with Jetpack Compose

# Clone the repository
git clone https://github.com/Saranyaaav/smart-traffic-analytics.git
cd smart-traffic-analytics

# Open Android project
cd frontend
# Open in Android Studio and sync project

Backend Setup

Ktor server with Kotlin

# Navigate to backend directory
cd backend

# Install dependencies
./gradlew build

# Run the server
./gradlew run
ü§ñ

AI Model Setup

Python-based AI processing service

# Navigate to AI directory
cd ai-processing

# Create virtual environment
python -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Run the AI service
python main.py

Google Cloud Deployment

Deploy to Google Cloud Platform

# Authenticate with Google Cloud
gcloud auth login

# Set project
gcloud config set project your-project-id

# Deploy to VM
gcloud compute scp --recursive . instance-name:~/app --zone=your-zone

Docker Deployment

Containerized deployment with Docker Compose

# Build and run with Docker Compose
docker-compose up -d

# Scale services

# View logs
docker-compose logs -f

Everything you need to know about the platform

The Smart Traffic Analytics Platform is built as an Android application using Kotlin and Jetpack Compose. It supports Android 7.0 (API level 24) and above. The backend services are platform-independent and can run on any cloud infrastructure.
Our AI-powered vehicle detection system uses YOLO v8 and achieves 95%+ accuracy in vehicle detection. It can process video streams at 30+ FPS and supports detection of cars, trucks, motorcycles, and buses in various weather and lighting conditions.
For development: Android Studio, JDK 11+, Python 3.9+, and Google Cloud SDK. For production deployment: Google Cloud Platform with custom VM instances, minimum 4 vCPUs, 16GB RAM, and 50GB storage. Docker support is available for containerized deployment.
